Due to the tree-node structure of the multi-agent system, at the tree root there is a conditional trigger for the 
at which root should the user command go to, as mentioned in the XR-MultiAgent project description. The "Complex Route" 
is designed when the user send out command that does not have clear spatial relationships involved in. For example, 
"re-organize the whole room layout, to change this (from a living room) to an office". 

To make this command work, there should be a collaboration among multiple agents (memory, asset, scene, verification 
and execution). The memory agent, or module, extracts information for what kind of layout will be a living room look like, 
so there is the image input. Also, due to the vagueness of the user's command, the memory module should also figure out what 
kind of assets should be involved in the office and there amount based on the layout prediction. 

Then, memory module pass these two information as JSON format to asset agent and scene agent. Asset agent first figure out the objects 
involved and their amount, and then append those from assets folder to the scene database. Scene agent figure out the spatial 
position of these newly added assets according to the layout prediction, verification checks the spatial coherence, and finally 
do the execution.
